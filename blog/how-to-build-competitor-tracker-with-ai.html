How to Build a Competitor Tracker with AI in Under 30 Minutes

You're spending hours every week manually updating competitive intelligence spreadsheets that nobody reads. Here's how to build a live, shareable competitor tracking page using Claude Code — no engineering background required.

The competitive intel problem every PMM knows

If you're a product marketer at a Series B SaaS company, you already know this ritual. Every Monday morning, you open six browser tabs — one for each competitor's pricing page. You check G2 for new reviews. You skim their changelog for product updates. Then you spend an hour or two updating a Google Sheet that your sales team glances at once a quarter.

It's not that competitive intelligence doesn't matter. It's that the format is broken. Spreadsheets go stale the moment you close the tab. PDFs get buried in Google Drive. Battlecards last until the next pricing change, which could be next week.

The real cost isn't the 3–5 hours per week you spend on manual tracking. It's that your sales team stops trusting the data because it's always a little bit out of date, and you stop investing in it because nobody reads it anyway.

There's a better way. And it takes about 30 minutes to set up.

What you'll build

By the end of this tutorial, you'll have a live HTML competitor tracking page that includes:

  • A pricing comparison table across your top competitors with tier breakdowns
  • A feature comparison matrix filterable by category
  • G2 ratings with review counts and recent sentiment
  • A changelog section showing each competitor's latest product updates
  • Responsive design that works on desktop and mobile
  • Filter controls so sales can drill into specific competitors

It's a single HTML file you can share as a link, host on Vercel for free, or drop into your company wiki. And because you built it with Claude Code, you can update it by just asking for changes in plain English.

Prerequisites

You need exactly one thing:

  • A Claude Pro subscription ($20/month) — this gives you access to Claude Code

That's it. You don't need to know JavaScript, HTML, or any programming language. You don't need a GitHub account (though it helps for deployment). You don't need to install a code editor. If you can type a sentence, you have the skills required.

Step 1: Set up Claude Code

Open your computer's terminal. On Mac, search for "Terminal" in Spotlight. On Windows, use PowerShell or Windows Terminal.

Install Claude Code with this command:

npm install -g @anthropic-ai/claude-code

If you don't have npm installed, you'll need Node.js first. Download it from nodejs.org — grab the LTS version and run the installer. It takes about 2 minutes.

Once installed, create a folder for your project and start Claude Code:

mkdir competitor-tracker
cd competitor-tracker
claude

You'll see a welcome message and a prompt. That's your conversational interface. Everything from here is plain English.

Step 2: Describe your tracker

This is where the magic happens. You're going to describe what you want the same way you'd explain it to a smart colleague who happens to be very fast at building things.

Here's the exact prompt you can use. Swap in your own company name and competitors:

Build me a competitor tracking page for our product marketing team.

We're Acme Analytics. I want to compare us against these competitors:
- Mixpanel
- Amplitude
- Heap
- PostHog

For each competitor, include:
1. Pricing tiers with actual prices (scrape their pricing pages)
2. Key feature differences vs us (focus on analytics, integrations, and reporting)
3. G2 rating and number of reviews
4. Their most recent 3 product updates from their changelog

Format it as a clean, professional HTML page with:
- A header with our company name and "Competitive Intelligence" as the subtitle
- Filter buttons to show/hide individual competitors
- Responsive tables that work on mobile
- A "Last updated" timestamp at the top
- Color coding: green where we win, red where we lose, gray for neutral

Make it look polished enough that I can share the link with our sales team.

Paste that in (with your own details) and press Enter. Claude Code will start working — you'll see it fetching data, building the HTML structure, adding styles, and creating the interactive filters.

The whole process takes about 15–20 minutes. You can watch it work or grab a coffee.

Step 3: Customize and extend

Once Claude Code finishes the first version, open the file in your browser to review it. Then iterate. This is the part that makes Claude Code fundamentally different from a traditional tool — customization is just another conversation.

Some examples of follow-up prompts:

> Add a section comparing our support options vs competitors.
  Include response time SLAs and support channels.

> Change the color scheme to match our brand. Primary: #2563eb,
  secondary: #f59e0b, background: #fafafa.

> Add a "Key Takeaway" callout box at the top summarizing where
  we're strongest and where we need to improve.

> Make the pricing section collapsible so it doesn't take up
  so much vertical space.

> Add Gong as a fifth competitor.

Each of these takes Claude Code a few minutes to implement. You're not writing code — you're having a conversation about what you want, and Claude Code handles the implementation.

Want to restructure the whole page? Change the layout from tables to cards? Add a dark mode toggle? Just ask. The barrier between "I want this" and "it exists" is a single sentence.

Step 4: Share with your team

You've got a polished competitor tracking page. Now get it in front of your sales team.

Option A: Deploy to Vercel (recommended)

If you have a Vercel account (free tier works), deploying takes one command:

npx vercel deploy

Vercel gives you a public URL you can share with anyone. Updates are just as easy — make changes with Claude Code, then run the deploy command again.

Option B: Share the file directly

The output is a single HTML file. You can email it, drop it in Slack, or upload it to your company wiki. Anyone can open it in a browser without installing anything.

Option C: Host on your internal tools

If your company uses Notion, Confluence, or an internal wiki, you can embed the page or link to the hosted version. Ask Claude Code to add your company's internal branding if needed.

Going further

The basic tracker is already more useful than the spreadsheet it replaces. But Claude Code lets you push further without adding complexity to your workflow.

Automate weekly updates

Ask Claude Code to add a script that refreshes the competitor data. You can run it manually every Monday, or set up a simple cron job to do it automatically. Here's what the prompt looks like:

> Add a Node.js script that refreshes all competitor data
  and regenerates the HTML page. I want to run it weekly.

Add Slack alerts

Get notified when a competitor changes their pricing or ships a major update:

> Add a Slack webhook integration that sends a message to
  #competitive-intel whenever a competitor's pricing changes
  or they publish a new changelog entry.

Build a G2 review dashboard

Go deeper on review intelligence:

> Create a separate page that tracks G2 review sentiment over
  time for each competitor. Show trending themes in positive
  and negative reviews from the last 90 days.

Connect to your CRM

If you export win/loss data from Salesforce or HubSpot, Claude Code can build analysis on top of it:

> I'm going to drop in a CSV of our closed-lost deals. Analyze
  which competitors we lose to most, the top 3 objections for
  each, and create a visual breakdown I can use in our QBR.

Each of these extensions takes 20–30 minutes. You're not learning a new tool or waiting on engineering. You're just describing the next thing you want.
